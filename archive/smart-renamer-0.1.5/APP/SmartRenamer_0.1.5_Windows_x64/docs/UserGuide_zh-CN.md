# SmartRenamer用户手册

版本：0.1.5  
平台：Windows x64  
发布日期：2026-06-10

## 1.程序说明

SmartRenamer是一个离线媒体批量整理工具，用于处理人物资源目录下的图片和视频文件。程序不会上传文件，不提供批量删除文件或目录功能。

主要功能：

- 递归扫描根目录下所有子文件夹。
- 重命名子文件夹，并追加或替换媒体统计后缀。
- 重命名子文件夹内的图片和视频文件。
- 清理子文件夹名称中的前缀、后缀、自定义文本和首尾空格。
- 使用智能规则或正则表达式清理文件夹名称中的无用片段。
- 执行前先预览，确认后才会真正改名。
- 支持冲突跳过、执行进度、操作日志和撤销上次重命名。

## 2.启动方式

双击运行：

```text
SmartRenamer_0.1.5_Windows_x64_Portable.exe
```

如果Windows出现安全提示，确认文件来自你的发布包后，可以选择继续运行。

## 3.基本流程

1. 点击文件夹按钮选择根目录。
2. 按需输入人物名称。
3. 选择排序方式。
4. 点击“扫描/Scan”。
5. 在预览区检查目标名称、警告和冲突。
6. 确认无误后点击对应操作按钮。

注意：程序不会重命名根目录本身，只处理根目录下所有层级的子文件夹和其中媒体文件。

## 4.人物名称输入规则

人物名称主要用于“文件夹重命名”。

- 如果只想重命名文件为纯序号，可以不输入人物名称。
- 如果输入人物名称，文件夹目标名称会优先规范为：

```text
[人物名称] - 主题 [统计]
```

示例：

```text
[Son Ye-Eun 손예은] - [BLUECAKE] BunnyLuXXX [99P 887MB]
```

如果原文件夹名称已经包含人物名称，程序会尽量避免重复添加。

## 5.文件夹重命名

文件夹重命名会递归处理根目录下所有子文件夹，不处理根目录本身。

统计规则：

- 只统计该文件夹当前层直接包含的图片和视频。
- 图片计为`P`。
- 视频计为`V`。
- 没有视频时省略`V`。
- 小于1GB显示MB，大于等于1GB显示GB，并保留两位小数。

示例：

```text
原文件夹名 [10P 1V 116MB]
原文件夹名 [108P 762MB]
原文件夹名 [18P 1V 1.92GB]
```

如果已有旧统计后缀，程序会替换旧后缀，而不是重复追加。

## 6.文件重命名

文件重命名会递归处理根目录下所有子文件夹内的图片和视频。

命名规则：

- 文件名只保留序号和原扩展名。
- 图片和视频分别独立编号。
- 编号只在文件实际所在的当前文件夹内生效。
- 默认使用文件名自然排序，也可以切换为修改时间或创建时间。

编号位数：

```text
1到9个：1.ext
10到99个：01.ext
100到999个：001.ext
1000到9999个：0001.ext
```

示例：

```text
001.jpg
002.jpg
003.jpg
001.mp4
002.mp4
```

当目标名称已经存在时，程序会跳过冲突项，并继续处理后续可重命名文件。

## 7.文件夹名称清理

文件夹名称清理会递归处理所选根目录下所有子文件夹，不处理根目录本身。

支持操作：

- 普通文本删除。
- 智能规则删除。
- 高级正则删除。
- 添加前缀。
- 添加后缀。
- 替换文本。
- 删除名称前后多余空格。
- 推荐并删除疑似无用内容。

所有清理操作都会先展示旧名称和新名称预览，确认后才会执行。

智能规则适合不会正则表达式的用户：

- 删除开头编号：删除`048 ARTGRAVIA_VOL252 #1`中的`048`。
- 删除Photo Vol编号：删除`DJAWA Photo Vol 0216`。
- 删除Vol编号：删除`Vol 0216`。
- 删除中间孤立编号：删除` - 048 ARTGRAVIA`中的`048`。
- 删除指定品牌词：输入`DJAWA`或`BLUECAKE`后删除对应品牌词。

高级正则适合需要自定义规则的用户：

```text
DJAWA Photo Vol \d+
^\d{2,4}\s+
\s+-\s+\d{2,4}\s+
```

正则小抄：

- `\d+`表示一串数字。
- `^`表示文件夹名称开头。
- `\s+`表示一个或多个空格。

示例：

```text
[Son Ye-Eun 손예은] - DJAWA Photo Vol 0216 - Staycation #5 [149P 2.99GB]
=> [Son Ye-Eun 손예은] - Staycation #5 [149P 2.99GB]

[Son Ye-Eun 손예은] - 048 ARTGRAVIA_VOL252 #1 [23P 33MB]
=> [Son Ye-Eun 손예은] - ARTGRAVIA_VOL252 #1 [23P 33MB]
```

## 8.进度、冲突和撤销

执行重命名时，界面会显示当前进度、已完成数量、跳过数量和失败数量。

安全规则：

- 目标重名时会跳过冲突项。
- 非法文件名会在预览中提示。
- 路径过长会在预览中提示。
- 最近一次成功执行的重命名操作可以撤销。
- 程序不提供批量删除文件或目录功能。

## 9.常见问题

### 扫描按钮为什么可以不输入人物名称？

从0.1.5开始，人物名称不是必填项。文件重命名只需要根目录和排序方式。

### 为什么文件夹有的会加人物前缀，有的不会？

如果输入了人物名称，程序会尽量补齐缺失前缀。如果原名称已经包含人物名称，程序会避免重复添加。

### 为什么图片和视频都从001开始？

这是当前版本的规则：图片和视频在同一文件夹内分别编号，互不抢占序号。

### 如果有冲突文件怎么办？

程序会跳过冲突项，并继续处理后面的文件。执行完成后可在日志中查看跳过记录。
